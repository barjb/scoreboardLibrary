---
name: generate-user-stories
description: Analyzes a system architecture and decomposes it into well-defined user stories with acceptance criteria, business value, dependencies, and implementation notes.
---

When invoked:
- Create directory `../../userStories` if it does not exist
- Create a new `.md` file for each user story
- Identify actors
- Generate user stories in a format
As a <user>
I want <capability>
So that <benefit>
- Add acceptance criteria
- Identify dependencies and prerequisites.
